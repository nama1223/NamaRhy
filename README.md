# NamaRhy

管楽器のための **リズム採点トレーニング** Web アプリ。カウントインのあとクリックに合わせて表示された楽譜のリズムで楽器を鳴らすと、発音タイミングの正確さ（早い／遅い）を色とミリ秒で表示・採点します。音程は問わず、タイミングだけを判定します。

単一の `index.html`（HTML + CSS + JS 全部入り、外部ライブラリなし）で動作します。

## 主な機能

- 🎼 **ランダムなリズム生成**（音楽的に自然な範囲）／**難易度30段階・変拍子あり**（4〜8小節、3/4・6/8・7/8・12/8など。テンポ・音価が段階的に変化、6連符まで）
- 🥁 **発音タイミング採点**：早すぎ／遅すぎを色分け＋ミリ秒で表示、総合スコアと傾向（走り／もたり）
- 🎯 **判定基準の切り替え**：「音の出始め」／「音のピーク」。録音後にワンタップで再採点可能
- ⚙ **キャリブレーション**：一定テンポの4分音符に合わせて鳴らし、端末と体感のズレ（平均オフセット）を「音の出始め」基準で補正（ピーク判定でも出始め基準を使用）
- 🎙 **マイク選択**：使用する入力デバイスを選択
- 📈 **プレイログ**（LocalStorage）：日時・楽譜・採点を保存し、過去のリズムに**再挑戦**して成長を確認
- 📱 スマホ1画面に収まる楽譜表示（[Bravura](https://github.com/steinbergmedia/bravura) 楽譜フォント使用）

## 使い方

マイクを使うため **https または localhost** で開く必要があります（ブラウザのマイク許可が必要）。

- ローカル確認: このフォルダで `python -m http.server` を実行し `http://localhost:8000/` を開く
- 公開: GitHub Pages 等の **https** な静的ホスティングに `index.html` と `Bravura.woff2` を置く

初回は **設定 → キャリブレーション** で端末との誤差を合わせると精度が上がります。

## 技術メモ

- 発音検出：マイク → **AudioWorklet** がブロックごとの RMS を送出 → メインスレッドでオンセット／ピーク検出（AudioWorklet 非対応時は AnalyserNode にフォールバック）
- クリックと検出はどちらも AudioContext の時計で計測するため、スケジュール時刻と直接比較可能。系全体のレイテンシはキャリブレーションで吸収
- 楽譜は1線リズム譜を SVG で拍位置に比例配置。符頭・旗・休符は SMuFL 楽譜フォント Bravura のグリフ、符幹・連桁・連符括弧は描画
